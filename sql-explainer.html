2. Le window function più usate

FunzioneCosa faQuando usarla
ROW_NUMBER()Numera le righe 1,2,3... senza pareggi"Prendi solo il primo/migliore per gruppo"
RANK()Numera ma lascia "buchi" in caso di pareggio (1,1,3)Classifiche dove i pareggi contano
DENSE_RANK()Come RANK ma senza buchi (1,1,2)Classifiche compatte
LAG(col)Valore della riga precedente"Confronta con il mese prima"
LEAD(col)Valore della riga successiva"Confronta con il mese dopo"
SUM() OVER()Somma cumulativa"Totale progressivo nel tempo"
AVG() OVER()Media mobile/di gruppo senza comprimere righe"Confronta ogni riga con la media del gruppo"
NTILE(4)Divide in N fasce uguali"Dividi in quartili"
Differenza chiave GROUP BY vs PARTITION BY
  • GROUP BY → comprime le righe, ne resta una per gruppo
  • PARTITION BY (dentro OVER()) → mantiene tutte le righe, calcola il valore rispetto al gruppo

JOINCosa restituisce
INNER JOINSolo righe presenti in entrambe le tabelle

7. L'ordine reale di esecuzione

Anche se scrivi le clausole in quest'ordine:

SELECT→ FROM→ WHERE→ GROUP BY→ HAVING→ ORDER BY→ LIMIT

Il motore le esegue in quest'altro ordine:

FROM→ WHERE→ GROUP BY→ HAVING→ SELECT*→ ORDER BY→ LIMIT

* qui vengono calcolate anche le window function

Questo spiega due cose fondamentali:
  • Perché non puoi usare un alias creato nel SELECT dentro il WHERE (il WHERE viene eseguito prima che l'alias esista)
  • Perché le window function richiedono una subquery se vuoi filtrarci sopra con WHERE (vengono calcolate nel SELECT, troppo tardi per il WHERE dello stesso livello)

8. Metodo per una domanda complessa

  1. Cosa voglio come risultato finale? Scrivilo in una frase.
  2. Devo comprimere righe (GROUP BY) o solo classificarle senza comprimerle (window function)?
  3. Mi serve un valore calcolato per filtrare dopo? → probabilmente serve una subquery o CTE.
  4. Sto unendo dati da più tabelle? → identifica la colonna in comune per il JOIN.
Scrivi prima la query "grezza" senza preoccuparti della perfezione, poi sistema cast, alias e ordine.
